1. What is SmartTrack?

MB SmartTrack is a full legal practice management dashboard built for Motsoeneng Bill (MB) law firm. It runs in the browser and manages every aspect of the firm's daily operations.

  • Automatic PC activity time-tracking and billing unit calculation (1 unit = 6 minutes)
  • Matter (legal file) management with notes, undertakings, and communications
  • Invoicing from tracked time with PDF generation and email delivery
  • Full Legal Practice Act-compliant trust accounting
  • Client FICA compliance monitoring
  • Document management — upload, tag, download
  • Shared firm calendar — meetings, court dates, deadlines, calls
  • HR management — staff profiles, leave, 360 feedback, payroll summaries
  • Client-facing self-service portal with OTP login

Tech stack: Next.js (React) frontend · Supabase (auth + database) · jsPDF for PDF generation · FNB/ABSA/Standard Bank/Nedbank CSV import for reconciliation.

Currency: South African Rand (R)  ·  Locale: en-ZA  ·  Billing units: 1 unit = 6 minutes (legal industry standard)

13. Client Portal (/portal)

Login flow

  1. Client enters email address
  2. System emails a 6-digit OTP code
  3. Client enters the code (6 digit boxes, auto-advances focus)
  4. Portal loads their data on success

What a client sees

  • Invoices — all their invoices with status (unpaid / partial / paid) and amounts
  • Trust account — all trust transactions for their matters
  • Matters — active matters and their status
  • Documents — documents shared with them (download any)
  • Upload documents — upload ID, proof of address, etc. directly to the firm
  • Service request — submit a request with description and urgency (normal / urgent / emergency)

New client request (unauthenticated)

Prospective clients with no account can submit a "Request Services" form with name, email, phone, ID number, service type, description, and urgency. These appear in the Manager's client requests queue.

Period management — month-end lock

Select a month (e.g. 2026-06). Toggle its status:

  • Open: Orange "🔒 Lock period" button — locks all entries for that month across the entire system
  • Locked: Red "Unlock" button with confirm dialog — should only be done with partner authorisation

When a period is locked, the Receipt, Payment, and Transfer forms all show "⚠ Period locked" and disable their Post buttons for any date in that month. All currently locked periods are listed on screen.

CSV import

Supports FNB, ABSA, Standard Bank, and Nedbank CSV exports. Parser handles DD/MM/YYYY and YYYY-MM-DD formats. Negative amounts = debits. Manual line-by-line entry is also available.

Side-by-side matching

  • Left column — Bank statement lines: Each line has a checkbox. Tick to mark as matched (greyed out with strikethrough). Delete button to remove a line.
  • Right column — System transactions: All posted receipts and payments. Checkbox to mark as matched.

Unmatched items on either side drive the difference shown in the summary cards.

Reconciliation certificate

"Print certificate" generates a formal document with: bank balance, system balance, difference, matched item count, period lock status, and two signature blocks (Bookkeeper + Partner/Director). Auto-triggers the browser print dialog.

23. Data Flow & Architecture

Client pays fees → Attorney posts Receipt → trust_transactions (type='receipt', status='posted')
Fees earned → Attorney posts Transfer → trust_transactions (type='transfer', status='posted')
Costs to pay → Attorney posts Payment → amount < R50k → status='posted' immediately
                                                      → amount ≥ R50k → status='pending' → Manager approves → status='posted'
Month-end → Bookkeeper imports bank CSV → matches → prints certificate → locks period
Audit → Reports tab → Trial Balance / Receipts / Payments / Transfers journals

Key technical facts

  • No stored balance column. Balances are calculated live by summing all posted receipts minus all posted payments per matter. This means the ledger is always accurate and cannot drift out of sync.
  • Period locks are stored in trust_period_locks as YYYY-MM strings. The application checks this on every form submit — locked periods block posting at the app level.
  • Receipt numbering is sequential: TRR-001, TRR-002… — calculated from the current MAX, no gaps.
  • Approval threshold is hardcoded at R50,000. All payments at or above this amount are always routed to pending, regardless of who submits them.
  • VAT on transfers: Trust holds VAT-inclusive amounts. Transfers deduct amount × 1.15 from trust; the firm keeps the net amount as professional fees.
  • Multi-branch: Every transaction records a branch_id. All reports, the trial balance, and the settings branch overview can be filtered per branch.
  • Refresh rates: Attorney trust auto-refreshes every 5 minutes. Manager trust refreshes every 30 seconds.
